(Fully compliant with SE2020 assignment.txt – pages 1-8)
Every member’s main entity now includes image upload (or image-related features) as per your request. This mirrors the Hostel example (Room has images, Booking links users/rooms) while keeping strong linkages across modules.
Core Linkage Flow (All Modules Interconnected):
Admin creates Product (multiple images) → User browses Products → Adds to Cart (with optional proof image) → Proceeds to Order (links products) → Makes Payment (optional receipt/screenshot image) → Delivery (optional delivery proof image by admin/driver) → User leaves Review (with optional review images) on the delivered Product.
All use Multer middleware (reusable, with file type/size validation: JPG/PNG < 5MB). Images stored via Cloudinary (recommended for live deployment) or local server folder → URLs saved in MongoDB. React Native uses Expo Image Picker (or react-native-image-crop-picker for multiple selection). Protected routes with JWT + role middleware. No hardcoded data.
Shared Group – Authentication Module (All test it; one lead can present):
User registration, login, bcrypt hashing, JWT, role-based access (“user” / “admin”).
